# FlowUp

팀이 막연한 아이디어를 문제 정의, 목표, MVP 기능, 역할, 개발 이슈로 연결해보는 실시간 협업 MVP 프로토타입입니다.

## 실행

```bash
npm install
npm run dev
```

프로덕션 빌드:

```bash
npm run build
```

## 구현된 범위

- 랜딩 페이지와 Hero 아이디어 입력
- 프로젝트 생성 및 localStorage 저장
- 아이디어 탐색 질문 진행
- 문제 정의 개인 의견 작성, 동시 공개, 공감, 댓글, 익명 투표, 최종안 확정
- 프로젝트 목표 저장
- MVP 기능 추가, 수정, 삭제, MVP 포함 전환, 가치/난이도 매트릭스 이동
- 팀원 역할 및 기능 담당자 배정
- MVP 기능을 개발 이슈로 변환하고 `할 일`, `진행 중`, `완료` 상태 변경
- 단계 완료 피드백 모달과 인터뷰 결과 화면
- 접속 팀원, 현재 단계, 최근 활동, 인터뷰 모드 패널
- 새로고침 후에도 프로젝트와 진행 데이터 유지

## 구현하지 않은 범위

실제 로그인/초대, 백엔드, WebSocket, AI API, 음성 채팅, 결제, GitHub 연동, 복잡한 권한 체계, 이후 잠금 단계의 편집 기능은 구현하지 않았습니다.

## 인터뷰에서 검증할 질문

- 단계별 질문이 빈 문서보다 아이디어 구체화에 도움이 되는가?
- 개인 작성 후 동시 공개가 토론의 질과 심리적 안전감에 영향을 주는가?
- 카드와 매트릭스가 우선순위 합의에 도움이 되는가?
- 투표 결과에서 최종 문제 정의와 MVP 기능까지의 연결이 자연스러운가?
- 어느 단계에서 가장 오래 머물거나 되돌아가는가?
